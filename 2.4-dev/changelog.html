Changelog for DPV 2.4

Published: 2026-02-25

This document is not accurate or up to date

This document describes the changes in DPV 2.4 release as compared to the previous DPV 2.3 release. This changelog covers the DPV specification as well as all extensions.

Summary

DPV 2.4 has backwards-incompatible changes regarding purposes where the meaning of concept(s) has changed. Fixes for these are simple to implement (see note further ahead in DPV concepts). Use of 2.4 is compatible with existing uses of 2.3 despite this. The changelog for 2.3 is available here.

  1. When updating are there are any issues? Yes, there may be a minor issue if using Service Provision purpose in DPV. The namespace for IEEE 7012 has been revised.see more
  2. Change in concepts: In total, DPV 2.4 and all its extensions contain 9982 concepts, and compared to 2.3 - 594 concepts have been added and 30 concepts removed. The count of added and removed concepts include changes to extension IRIs and moving concepts between extensions. see more
  3. Development process: The development process for DPV is carried out through GitHub and meetings. Participation is open and welcome. see more
  4. Contributors: list of contributors to version 2.4. see more

Do I need to update or change something?

You should consider updating to the latest version of the DPV as it provides a more comprehensive set of terms, has new extensions with refinements and advances in representing information, and represents more recent and up to date knowledge within the domain.

There is one breaking change between 2.4 and 2.3 where the DPV concept ServiceProvision has been made narrower i.e. it now represents a subset of previously possible things. The new concept ServiceMaintainence is the broader concept and comes with several granular concepts to better describe service related concepts. Existing users of ServiceProvision can continue to use the concept, but are recommended to additionally evaluate whether their use should also include one or more of the newly added concepts to correctly describe the intended purposes. There are no changes to the underlying model and semantics.

The namespace for the IEEE 7012 extension has changed following a change in its name from P7012 to 7012, and the new IRI as /standards/ieee/7012. Adopters are advised to switch to this namespace and concepts in it if using the draft extension.

Going ahead, we do not foresee major breaking changes in the future release which is tentatively labelled 2.4 and scheduled to be released in July 2026.

The name and namespace of the extension has been changed to accurately reflect the formal terminology used by IEEE i.e. "p7012" reflected the draft standard while "7012" reflects the published standard. The namespace has also been changed to be /standards/ieee/7012 to better differentiate between different standards development organisations and their specific numbered standards. As a result of this, all concepts have been shown as removed from the old namespace and added to the new namespace.

Development Process

  1. GitHub: All code and artefacts are hosted in a github repo, and it is also used for discussions via issues and for hosting a community wiki.
  2. Git branches: Development takes place on the dev branch, and the development version is suffixed to indicate ongoing work as 2.4-dev. The main branch hosts stable and published releases and is updated for fixes and for storing working notes and minutes for the DPVCG meetings.
  3. Announcement for releases: The issue #145 acts as a 'sticky' board for announcements regarding releases, fixes, and other relevant news. Adopters can subscribe to this issue to be notified.
  4. Improved guides and tutorials: The DPVCG plans to provide more/better guidance to support the adoption and use of DPV. A tentative list can be found at the guides page and requests for additional specific content or guidance can be made via github.
  5. Multilingual support: This feature is planned for future releases where the DPV concepts will be provided with labels, definitions, and notes in more languages. Planned languages include Italian, French, German, and Spanish. If you are interested in having another language added, please discuss in issue#89.
  6. Release schedule: The DPVCG plans to provide two releases per year to enable rapid incorporation of changing regulations and facilitating adoption.
