Roadmap for Guides

<ghurlbot> Issue 242 Provide Guides for Supporting Adoption & Use of DPV (by coolharsh55)

BeatrizEsteves: Priority is to finish off some of the currently proposed guides by the release of 2.3

see current issues for guides: https://github.com/w3c/dpv/issues?q=is%3Aissue%20state%3Aopen%20guide

DPIA - A paper can be adapted - "A Semantic Specification for Data Protection Impact Assessments (DPIA)" https://doi.org/10.3233/SSW220007

PaulRyan: Perhaps Delaram and Tytti will have interest in collaborating for this one.

ROPA - Paul's PhD work can be adapted here, DPCat specification using DCAT should be discussed separately. The guide will focus on showing which concepts from DPV are useful for which ROPA field. This will be aligned with implementation of the ISO/IEC 27560 revision (which as PII Processing Records and Privacy Receipts instead of just for consent)

29184 Privacy Notices - Harsh has a working draft https://harshp.com/dpv-x/guides/notice-29184/ based on consent guide

BeatrizEsteves: +1 to add it on the next release

JulianFlake: Is there documentation on how it relates to ISO 27560 on consent records?

Data Breach - A paper can be adapted - "Towards a Semantic Specification for GDPR Data Breach Reporting" https://doi.org/10.3233/FAIA230956

Rights Exercise - A paper can be adapted - "How to Manage My Data? With Machine--Interpretable GDPR Rights!" https://doi.org/10.3233/FAIA241254

BeatrizEsteves: I would like to improve the current draft guide by DEC

Risk Assessment - needs to be written from scratch, Harsh can do this quickly as he has written most of the RISK extension content

ODRL - https://dev.dpvcg.org/guides/dpv-odrl has a draft mapping table but is missing a github issue (Beatriz to check with ODRL CG on how to implement this correctly?)

BeatrizEsteves: seems doable by 2.3, I have already shared the current state with ODRL CG and have feedback to improve it.
