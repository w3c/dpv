DPVCG Charter

<ghurlbot> Issue 399 Establish DPVCG Charter (by coolharsh55)

HarshPandit: Working document: https://docs.google.com/document/d/1poo7rYNBlkd4Ag0YHB_il2Xe2hwtHWfROgl5WgrNgyc

HarshPandit: (discussing structure of the document, explanation of each section, why the Charter is needed and what is its role) We have to write and discuss three sections: Goals, Scope, and Out of Scope

HarshPandit: Example Solid CG Charter https://www.w3.org/community/solid/charter/

ArthitSuriyawongkul: What about ethical web principles mentioned in the Solid charter? Do we do something similar in ours? https://www.w3.org/TR/ethical-web-principles/

HarshPandit: No, because these are written for a web technology in mind, which Solid is, whereas DPV isn't. What would be more relevant is the Privacy Principles https://www.w3.org/TR/privacy-principles/ which talk about user control and consent/opt-in-out etc. which we can state as being relevant. In the future we can work on mapping how privacy principles can be implemented using DPV

ArthitSuriyawongkul: I would argue the ethical web principles are also about the web as in semantic web

HarshPandit: Agree, my concern is that if we specify ethical web principle we might be saying DPV is only for the "web" so otherwise we have broader principles.

HarshPandit: So we have privacy preferences in scope for our charter, and then ethical principles in Goals as an example of what DPV can be helpful for as an application

ArthitSuriyawongkul: Agreed that the privacy principles is much more focused

HarshPandit: For drafting the charter, who volunteers to support this?

ArthitSuriyawongkul: +1

ACTION: Set up DPVCG Charter draft -- Harsh and Art
