Agents

GeorgKrog: (paraphrased) use of Agents to perform tasks in workflows

ArthitSuriyawongkul: better to call it AgenticAISystem if its to be a subclass of AI System

ArthitSuriyawongkul: how is this different from AI Agent? For an Agent, you specify some goals or response to conditions and it goes off to do the work without response from the user. For my Agentic AI is that you define goals or results but not the how to achieve that, but the user is still waiting e.g. for an hour. So there is an element for waiting for the response.

HarshPandit: Agentic AI involves use of AI agents to execute tools/workflows

ArthitSuriyawongkul: so in thise sense Agentic is an adjective to the AI System to state AI agents are being used to achieve the tasks

next steps: look for normative definitions, otherwise discuss only adding the basic concepts for the vocabulary without additional specifics for v2.3

AI Development Phases

<ghurlbot> Issue 294 AI Model Development Phases (by coolharsh55)

HarshPandit: these are not well defined/standardised as compared to the ISO 5338 lifecycle; so should we model them and how do we select the concept?

DelaramGolpayegani: depends on which steps you want to take to develop the model; Should these concepts also consider whether it is a GenAI model and what are the steps to integrate that into the organisation

HarshPandit: these are processes as they involve e.g. data collection ...

ArthitSuriyawongkul: will this include the domain or user etc. before data collection. This is like a requirement.

DelaramGolpayegani: This is like requirements engineering in software development. How much of the lifecycle do we want to consider, and how many names/labels we want to include for different stages. This could be useful for looking at the value chain and seeing who is involved and when/where. E.g. to see who was involved in the earlier stages and tracing what happened/who is responsible. E.g. using PROV-O.

HarshPandit: sounds like the concepts are useful; so should these be done as part of existing lifecycle https://dev.dpvcg.org/2.2/ai/#vocab-lifecycle or separately?

DelaramGolpayegani: makes sense to start with lifecycle and see if it fits and then look for a different place if not; this will be helpful for the data governance and implementation of the AI Act

ArthitSuriyawongkul: reason to think about this is that the previous Biden gov executive order required documentation about stakeholders during the development of the AI requirements procedures; though this executive order is no longer around

DelaramGolpayegani: option to go for a sub-extension of AI for lifecycle

HarshPandit: no, because this fits in AI

next steps: develop a proposal based on ISO 5338 and aligned with common terminology used in AI domain
